* Deadly simple commit message generator on `gptel`
Generate git commit messages automatically using [[https://github.com/karthink/gptel][gptel]].

This package integrates [[https://github.com/karthink/gptel][gptel]] with git to automatically generate meaningful commit messages based on the staged changes (diff) without requiring user interaction.

* Features

- Automatic commit message generation from git diffs
- Zero-prompt interface - generates messages minimum user interaction
- Respects gptel configuration and model selection
- Configurable diff size limits to manage API usage
- Support for both staged changes and committed diffs
- Exclude files from the diff with configurable globs
- Follows conventional commits format

* Installation
- Emacs 30.1 or later

** Manual

1. Clone this repository or download ~gptel-commit-message.el~
2. Add to your Emacs configuration:

#+begin_src elisp
(use-package gptel-commit-message
  :load-path "/path/to/gptel-commit-message"
  :after gptel)
#+end_src

* Usage

** Interactive Commands

*** =gptel-commit-message-generate=

Generate a commit message and insert it into the current buffer at
point. Internally it uses gptel's streaming request support
asynchronously. While the response is streaming, partial text is
inserted at point using a temporary face, with an animated indicator at
the end of the temporary text, and then normalized once the request
completes. If generation cannot be started, it returns nil and
stores the message in
~gptel-commit-message-last-error~.

#+begin_src elisp
  ;; just call it from M-x
  (gptel-commit-message-generate)

  ;; Or define keybinding for that.
  (define-key
   git-commit-mode-map "C-c g" #'gptel-commit-message-generate)
#+end_src

** Customization

*** =gptel-commit-message-prompt=

Customize the prompt sent to the LLM:

#+begin_src elisp
(setq gptel-commit-message-prompt
      "Generate a concise git commit message following conventional commits format. 
       Consider the changes and write ONLY the commit message.")
#+end_src

*** =gptel-commit-message-max-diff-size=

Limit diff size to manage API costs (default: 50000 characters):

#+begin_src elisp
(setq gptel-commit-message-max-diff-size 30000)
#+end_src

*** =gptel-commit-message-use-staged-changes=

Use staged changes (default: t) or already-committed changes:

#+begin_src elisp
;; Use staged changes
(setq gptel-commit-message-use-staged-changes t)

;; Use HEAD~1..HEAD instead
(setq gptel-commit-message-use-staged-changes nil)
#+end_src

*** =gptel-commit-message-backend=

Specify a particular gptel backend (optional). If not set, uses =gptel-backend=:

#+begin_src elisp
(setq gptel-commit-message-backend
      (gptel-make-openai "ChatGPT"
        :key "your-api-key"
        :models '(gpt-4)))
#+end_src

*** =gptel-commit-message-excluded-globs=

Exclude matching files from the diff sent to the LLM.
The default value excludes common lockfiles:

#+begin_src elisp
(setq gptel-commit-message-excluded-globs
      '("*.lock" "*-lock.*"))
#+end_src

Add more patterns by extending that list:

#+begin_src elisp
(setq gptel-commit-message-excluded-globs
      '("*.lock" "*-lock.*" "dist/**" "vendor/**"))
#+end_src

Patterns are passed to git as glob pathspec exclusions relative to the
repository root.

* How It Works

1. **Fetch Diff**: Retrieves the git diff from staged changes (or committed changes if configured), excluding any matching globs
2. **Truncate if Needed**: Respects size limits to avoid excessive API usage
3. **Send to LLM**: Sends the diff and prompt through gptel's streaming request API
4. **Preview**: Streams partial output into the current buffer using a temporary face and animated indicator
5. **Finalize**: Replaces the preview with the finalized message when the request completes

* Troubleshooting

** "No gptel backend configured"

Ensure gptel is installed and configured:

#+begin_src elisp
(use-package gptel
  :config
  (setq gptel-backend
        (gptel-make-openai "ChatGPT"
          :key gptel-api-key)
        gptel-model 'gpt-4))
#+end_src

** Large diffs being truncated

Adjust the limit:

#+begin_src elisp
(setq gptel-commit-message-max-diff-size 100000)
#+end_src

** Generated messages are too verbose

Customize the prompt:

#+begin_src elisp
(setq gptel-commit-message-prompt
      "Generate a VERY SHORT commit message (max 50 chars). Format: type: subject")
#+end_src
